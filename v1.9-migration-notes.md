1. **D10 is NOT a pipeline stage** — it's a retrieval service consulted from D1, D2.5, D7, D9 at their respective stages, in three modes (Direct / Consult / Validate). No `corpus_check` slot. Verbatim guard is D10 output post-processing, not a pipeline gate.
2. **Storage architecture**: Ignore Cloudflare R2 + Chroma. Use Express + MySQL + Forge. D10 embeddings behind `VectorStore` interface backed by JSON-array vectors in MySQL with SQL cosine similarity (Wave 2.5). Queued migration to Chroma/pgvector in Wave 4 when corpus > ~5K chunks.
3. **Greenfield episode_state**: No production users. New episodes use v1.9 17-stage flow. Legacy 12-stage records marked read-only or migrated lazily.
4. **Agent designations per Section 7B**: D7 = FX Compositor (NOT music_scoring). D8 = Voice Director Critic (NOT sfx_foley). D9 = Sakufuu Tracker (NOT audio_mix). Music and SFX produced by infrastructure (MiniMax Music via Replicate, FX library overlays) within X-Sheet / Ato-Fuki / FX Pass / Mastering stages — no own D-designations.
5. **Section 4 numbering quirk**: Blueprint header says "v1.1" but body references v1.3, v1.4, v1.8 features. It IS the canonical v1.9 spec.
6. **Tables in Section 7B did not extract cleanly**: Corpus listings blank in this extract. Not required for stage migration.

## Section B — Blueprint v1.9 Section 4 (Verbatim Extract)

### Stage 1 — Script (脚本 / Kyakuhon)
- AI generates draft script from manga source or creator brief. User edits and approves. First gate; nothing downstream begins until user signs off.
- [v1.4] D9 Sakufuu Tracker reads this stage's outputs and updates user's prompt-style adapter.

### Stage 2 — Anime Type Selection
- User selects anime category (Shōnen / Shōjo / Seinen / Josei / Kodomomuke). Each selection bundles a LoRA, prompt-template family, default frame-rate convention, music-bed mood vector, and [v1.3] a default effect-density profile that seeds D1's ekonte effect tags.
- AI may suggest category based on script tone; human owns final decision.

### Stage 3 — Character Sheets (キャラクターデザイン / Kyarakutā Dezain) [NEW]
- Promoted to top-level stage in v1.4. D0 Character Designer generates complete reference dossier per character: full-body view (front, three-quarter, side, back); face close-up at canonical expression; expression sheet (neutral, smiling, sad, angry, surprised, pained, determined); signature outfit detail sheet; cybernetic/fantasy/unique features documented separately.
- D0 Self-Critic Role [v1.4]: D0 reviews generated sheets before presenting to user, flagging cross-character distinguishability, internal consistency across views, outfit feasibility for active video model, genre alignment, and missing artifacts. User approves each character before storyboard begins. Blocked gate — no path forward without sheet approval.
- After multiple episodes, D9's sakufuu document conditions D0's outputs so character variants align with user's signature design preferences.

## Section B — Blueprint v1.9 Section 4 (continued from pages 6-10)

### Stage 4 — Color Script (色彩設計 / Shikisai Sekkei)
- D6 Color Director produces episode-wide palette spec: per-character palette swatch, per-scene palette swatch, time-of-day modifier curve.
- User reviews and locks specific colors. Every downstream stage references locked palette.

### Stage 5 — Storyboard (絵コンテ / Ekonte)
- D1 produces richer metadata than earlier versions. Core panel metadata: shot type, camera angle, camera movement, duration, dialogue placement, character attribution, SFX cue tags, emotion tag, music + lip-sync, transition tags.
- [v1.3] Transition vocabulary: cut, フェードイン/フェードアウト (fade), cross-dissolve/クロスディゾルブ, dip to black/white, BG置き換え/BG-okikae, 波ガラス/wave glass.
- [v1.3] In-shot effect tags: ガブレ (gabure) = frame shake/judder; 画面動 (gamen-dou) = whole-frame movement; bokeh-pull/ボケ = focus shift; 光角 (koukaku) = light effect; 波ガラス (wave glass overlay) = in-shot distortion.
- D1 Critic on Effect Usage [v1.3]: flags overuse, genre mismatch, emotional gap.
- **Manga Branch**: v1.8 makes ekonte a publishable manga artifact. Can branch to Stage 5.5 (Manga Finishing) or continue to anime (Stages 6+).

### Stage 5.5 — Manga Finishing (漫画仕上げ / Manga Shiage) [NEW, branch]
- D10.M Manga Craft Sensei leads in Direct mode. Applies canonical manga finishing conventions: screentone, ink-line discipline, dialogue bubble placement, page composition, cover design.
- Output: print-ready PDF at `awakli-episodes/{episode_id}/manga/`. Cost: ~$2-5/volume.

### Stage 6 — Layout (レイアウト / Rēauto) [NEW]
- D1.25 Layout Director: composition (rule-of-thirds, focal-point), perspective (vanishing-point lock, impossible geometry flags), character scale consistency, camera-move arrows, background vs character placement.
- User can swap angles, reframe, request alternatives. Approved layout conditions every genga call.
- Cost: ~$0.05/panel × 19 panels = ~$1.00.

### Stage 7 — Key Animation (原画 / Genga)
- D1.5 Genga Director: 2-4 key poses per shot (start, mid-action, end, optional climax). 19 slices → 50-75 genga images, ~$1/episode.
- Conditioned on: character reference sheet (Stage 3), color script (Stage 4), approved layout (Stage 6), previous genga keyframe in sequence.
- Flip-book preview: slideshow with timing indicators (hold 0.8s, transition 0.3s).
- **Genga Branch**: v1.8 — approved genga is publishable artifact (Pro+ tier). Can route to Stage 7.5.

### Stage 7.5 — Genga Finishing (原画仕上げ / Genga Shiage) [NEW, branch]
- D10.G Genga Craft Sensei leads in Direct mode. Cleanup, line-weight discipline, on-model adherence, color script application, plate/timing markers, master-style references.
- Output: print-ready file at `awakli-episodes/{episode_id}/genga/`. Cost: ~$3-8/genga set.

### Stage 8 — Sakuga Kantoku Review (作画監督) [NEW]
- D2.5 Sakuga Kantoku: single Opus pass over entire approved genga set. Character consistency, style coherence, continuity, color script compliance.
- Punch list with proposed fixes. User resolves each item. Only after kantoku sign-off does genga proceed to video gen.
- This is the missing role that lets character drift slip through in P13.

### Stage 9 — Sakuga Tagging (作画指定) [NEW]
- User marks each shot as standard or 作画 (sakuga). Sakuga shots get 5-7 keyframes (vs 2-3) and route to premium video-generation tier.
- AI may suggest sakuga candidates based on emotion arc; human owns budget allocation.

### Stage 10 — In-Betweening (動画 / Douga) — Video Generation
- Approved genga → video model. With layout + keyframes as conditioning, task collapses to interpolating motion between known endpoints (image-to-video).
- **One-Model-Per-Tier Rule** [v1.2]: single model per tier within episode. No mixing models.

### Stage 11 — Per-Clip Continuity Check (D5.5) [NEW]
- Each clip checked before assembly. 3 keyframes extracted (start/mid/end) from individual clip. CLIP similarity against character reference sheet + multimodal Opus check for character/outfit/style/script match.
- Failures → slice regeneration. Retry budget: 3-5 takes at ~$0.20/re-roll.
- **Correctly placed pre-assembly** (not post-subtitles as my first mapping had it).

### Stage 12 — Timing Chart (タイムシート / X-Sheet) [NEW]
- Frame-accurate timeline. User marks dialogue start/end per character, camera/cut timings, hold frames vs motion frames, BGM cue points, SFX hits.
- X-sheet seeds Stages 13 (ato-fuki) and 14 (FX pass). Subsumes legacy music_scoring + sfx_foley.

### Stage 13 — Post-Recording (アフレコ / Ato-Fuki)
- TTS dialogue generated to X-sheet timing. Dedicated lip-sync model receives tight timing windows from X-sheet.
- User can re-record any dialogue line without re-rendering video.
- Renamed from voice_synthesis.

### Stage 14 — FX Pass (特殊効果 / Tokushu Kōka) [NEW]
- D7 FX Compositor reads ekonte's 光角, 波ガラス, aura, lighting tags. Routes to FX library or generation path.
- Genre-aware: Shōnen gets energy auras, speed lines; Shōjo gets 光角, sparkles; Seinen gets atmospheric haze.
- FX library is anime-type-aware (Stage 2 selection determines active assets).

### Stage 15 — Final Compositing (撮影 / Satsuei)
- Combines character video, backgrounds, FX layers, camera moves, audio into finished cut.
- Music bed and SFX layered against X-sheet cue points. Applies remaining ekonte effect tags (ガブレ, 画面動, bokeh-pull, transitions).
- Renamed from video_composite.

## Pages 11-15: Stages 16-17 + Section C (Blueprint Section 7B)

### Stage 16 — Mastering & Hybrid Harness
- Existing mastering pass (target -16 LUFS, -1.5 dBTP) + Hybrid Harness (H1 + D5 + H2 + Regeneration Executor).
- Two new additions: **audio coverage check** (H1 fails if audio shorter than video by >0.1s), **card legibility check** (H1 measures contrast/brightness on title/end cards + one-frame OCR pass).
- Subsumes legacy audio_mix + subtitle_render + episode_publish.

### Stage 17 — Continual Learning (LoRA + Sakufuu) [NEW]
- After kantoku final sign-off, training jobs enqueued for:
  1. **Per-Character LoRAs** (every episode) — trained on approved character sheets + genga + keyframes. $2-10/character/episode.
  2. **Sakufuu Aesthetic LoRA** (every 3-5 episodes) — user-level LoRA trained from approved frames across all characters/episodes.
- D9 Sakufuu Tracker maintains three layers:
  1. **Sakufuu Document** — live JSON profile (effect-frequency, composition preferences, pacing, voice-direction, sakuga budget, override patterns per stage). Injected into every D-agent's context.
  2. **Sakufuu Aesthetic LoRA** — user-level visual signature LoRA. Trained every 3-5 episodes.
  3. **Prompt-Style Adapter** — learns user's preferred prompting language and reformats AI outputs to match.
- **Three-Way Adapter Composition** at runtime: Anime-type LoRA (genre) + Character LoRA (identity) + Sakufuu LoRA (creator identity). Blend weights tuneable.

### Section C — Blueprint v1.9 Section 7B: D-Agent Designations

#### D10 Craft Sensei Constellation
- **D10 is NOT a pipeline stage** — it's a retrieval service (sensei, not producer/critic).
- v1.8 expands D10 into three sub-senseis sharing retrieval infrastructure but distinct corpora:
  - **D10.A** — Anime Craft Sensei (existing scope)
  - **D10.M** — Manga Craft Sensei [NEW, v1.8]
  - **D10.G** — Genga Craft Sensei [NEW, v1.8]
- Three modes of engagement: **Direct** (D10 leads the stage), **Consult** (D10 provides context, upstream agent decides), **Validate** (D10 reviews for craft compliance).

#### D10 Per-Stage Plug-In Points (NOT pipeline slots)
- Stage 1 (Script) — D10.A Consult mode for genre-appropriate dialogue/cadence
- Stage 5 (Ekonte) — D10.A Consult mode for panel-rhythm/effect-density; D10.M Direct mode for manga-track ekontes
- Stage 5.5 (Manga Finishing) — D10.M Direct mode
- Stage 7 (Genga) — D10.G Direct mode for animator-style references
- Stage 7.5 (Genga Finishing) — D10.G Direct mode
- Stage 8 (Sakuga Kantoku Review) — D10.A and D10.G Consult mode
- Stage 14 (FX Pass) — D10.A Consult mode for genre-appropriate effect deployment
- Stage 17 (Continual Learning, D9) — D10.A Consult mode for sakufuu interpretation against canonical traditions

#### D10 Storage Architecture
- Blueprint specifies Cloudflare R2/AWS S3 + local Chroma + Bloom filter + cold archive (S3 Glacier).
- **Override for Awakli build:** Express + MySQL + Forge stack. D10 embeddings behind `VectorStore` interface backed by JSON-array vectors in MySQL with SQL cosine similarity (Wave 2.5). Planned migration to Chroma/pgvector in Wave 4 when corpus exceeds ~5K chunks. Bloom filter is in-process. Forge proxy storage is the blob layer.

#### D10.M — Manga Craft Sensei [NEW]
- Manga-specific corpus (different art form from anime). Plug-in points: Stage 5 (Ekonte), Stage 5.5 (Manga Finishing), manga publishing flow.
- Master-aware: recommends canonical conventions from relevant master (Toriyama for Shōnen, Asano for Seinen psychological).
- Estimated corpus cost: ~$600-1,000 in printed books + free public sources.

#### D10.G — Genga Craft Sensei [NEW]
- Genga-specific corpus (key-animation art form). Plug-in points: Stage 7 (Genga), Stage 7.5 (Genga Finishing), genga publishing flow, Stage 8 (Sakuga Kantoku Review).
- Distinctive capability: animator-style reference (Imaishi energy, Nakamura physicality, Sadamoto refined linework).
- Estimated corpus cost: ~$1,000-3,000 (genga collection books are expensive).

#### Pro-Tier Creator-Uploaded Reference
- Pro creators can upload private influences as supplement to base Craft Library. D10 references these alongside base library for that user's projects.

#### Verbatim-Output Guard
- D10's output is post-processed: split into 15-grams, compared against Bloom filter of all source corpus 15-grams. Any match → response rejected, D10 asked to rewrite.
